## Path rule suggestions — consistent look when re-opening a configured rule / 路径规则建议——重新点击已配置的规则时外观一致

**English**

- **Fixes a follow-up to 1.0.11's folder browsing**: previously, a brand-new (empty) rule row showed the new level-by-level folder browser, but once you'd set a row to `/` or a folder like `Notes/` and clicked it again, the popup fell back to the old flat "matches everything" list — the look was inconsistent with the browser.
- **Now**: clicking a rule that's already set to the root `/` or a real folder keeps the hierarchical browser and opens **at that level** (e.g. clicking `Notes/` shows what's inside `Notes/`, with the header confirming where you are and `⬅` to go back up). Typing a partial name still switches to whole-vault fuzzy search, exactly as before.

**中文**

- **修复 1.0.11 文件夹浏览的一个遗留问题**：此前，新增的空白规则行会显示新的「按层级浏览文件夹」外观，但一旦你把某行设成 `/` 或某个文件夹（如 `Notes/`）后再次点击它，弹窗又会回落成旧的「匹配一堆」的扁平列表——外观与浏览器不统一。
- **现在**：点击已经设为根 `/` 或某个真实文件夹的规则行，仍保持分层浏览，并**定位到该层级**（例如点 `Notes/` 会展示 `Notes/` 里的内容，顶部标题确认你所在的位置，`⬅` 可返回上一级）。输入片段文字仍会切回全库模糊搜索，和以前完全一样。
