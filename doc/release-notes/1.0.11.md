## Path rule suggestions — hierarchical browsing + a display glitch fixed / 路径规则建议——分层浏览 + 修复一个显示故障

**English**

- **Browse folders level by level**: click into an empty path field and you'll see the vault root's folders and files; click a folder's small ▸ arrow to step into it and see its contents, one level at a time — no more scrolling through every file in the vault at once. Clicking a folder's name still selects it as the rule immediately, same as before.
- **Jump back up, or pick the current folder**: while browsing, click the path shown at the top of the list to use that folder itself as the rule, or click the ⬅ arrow to go back up one level.
- **Type to search still works exactly as before**: once you start typing, the popup switches back to the familiar fuzzy search across the whole vault.
- **Fixed**: after picking the root rule ("/") and clicking the field again, the suggestion list used to show a confusing, seemingly-broken result (top-level folders vanished, replaced by unrelated deeply-nested items). That's fixed — the root option no longer interferes with the search.

**中文**

- **按文件夹层级浏览**：点击空白的路径输入框，会先看到仓库根目录下的文件夹和文件；点文件夹右侧的小箭头 ▸ 可以进入该文件夹继续往下看，一层一层来，不用再面对一次性铺开的全库文件列表。点文件夹名字本身仍然是立即选中它作为规则，和以前一样。
- **返回上一级，或直接选中当前文件夹**：浏览时点列表顶部显示的当前路径，可以把这个文件夹本身选作规则；点 ⬅ 箭头则返回上一级。
- **打字搜索完全不受影响**：一旦开始输入文字，弹窗照常切回熟悉的全库模糊搜索。
- **修复**：选中根规则「/」之后再次点击输入框，建议列表以前会出现看起来很诡异的内容（顶层文件夹全部消失，换成一堆不相关的深层文件）。现在已经修好，根选项不会再干扰搜索了。
