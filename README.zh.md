# Auto Headings（自动标题编号）

[English](README.md) | **简体中文**

为 [Obsidian](https://obsidian.md) 自动编号 Markdown 标题——而且经得起编辑、改名、调整结构的折腾。

## 这个插件解决什么问题

**在长文档中间插入一节，后面所有编号全错了。** 一份 40 个标题的规格文档，或一学期的课堂笔记，手动把 `2.3` 改成 `2.4`、`2.5`……又慢又容易出错。Auto Headings 每次你停止输入后自动重新计算整个文件的编号。

**改了一个已编号标题的名字，所有指向它的 `[[笔记#标题]]` 链接全断了。** 大多数编号插件只更新数字，链接文字却留在原地不动。Auto Headings 的 Backlink 同步会在同一次编辑里改写库中所有相关引用——哪怕你连标题文字一起改，链接照样能解析。

**一种编号风格满足不了整个库。** 会议纪要想要 `1 / 1.1 / 1.1.1`；书稿想要「第一章」「一、」；学术论文想让"目录""参考文献"完全不占编号。Auto Headings 让每个文件夹、甚至每个文件都能用自己的模板，配合白名单把不该编号的标题彻底排除在计数之外。

装上即得 `1` / `1.1` / `1.1.1` 多级编号，零配置。然后按需定制，多细都行。

## 开箱即用

下面这些是插件启用之后立刻就有的，你还没打开一次设置面板。

-   **零配置。** 装上、打开笔记、开始编辑，H2 及更深的标题立刻获得 `1` / `1.1` / `1.1.1` 编号——不用任何设置。
-   **你的标题层级神圣不可侵犯。** 插件只增删编号前缀，**绝不**改写 `#`/`##`/`###`。一个文件里有多个顶级 `# H1` 时插件不动它们，只把每个 H1 当作分节边界（其下编号各自重新开始）。
-   **不跟你抢打字节奏。** 重新编号在你停止输入后按防抖延迟触发（50–2000 ms，可调），并以**一次编辑器事务**重写整个文件——一次 `Ctrl/Cmd+Z` 即可撤销，光标位置也保持不动。
-   **无论库多大都没有后台开销。** 插件**只**解析和重写你正在编辑的那一个文件——由编辑、手动命令、或设置面板改配置后对活动文件重排三种时机触发，**从不**为"用了某模板的所有文件"扫描全库。没打开的文件永远不会被碰，模板/路径规则再多，空闲时也不产生任何开销。
-   **两层独立开关，外加一个手动兜底。** 启用插件不等于让它在整个库里随便动手：一个全局的自动编号开关决定编辑笔记是否会触发重新编号。在笔记 frontmatter 里加 `obsidian-auto-headings: true` 或 `false`，可以针对这一个文件覆盖该开关——让它在全局关闭时单独常驻开启，或者让它在全局开启时对某个文件完全免打扰。**立即重新编号**命令把这两层都绕开：你主动敲了命令，意图已经足够明确，直接跳过一切开关立刻编号。
-   **配置只住在设置里，不住在你的笔记里。** 模板、路径规则、白名单全部在设置面板里管理——除了上面那一个可选的开关键，frontmatter 里不会被写入任何东西。打开一篇笔记，看不出插件装没装。
-   **开箱即双语。** 整个界面自动跟随 Obsidian 自身的语言设置（中文 / 英文），也可以手动锁定某一种。移动端完全可用（`isDesktopOnly: false`）。

### 快速上手

1. 安装并启用插件（见[安装](#安装)）。
2. 打开任意笔记开始编辑——H2 及更深的标题自动获得编号。
3. 想再深入定制，打开 **设置 → Auto Headings**：
    - **全局设置**：语言、全局自动编号开关、Backlink 同步、防抖延迟。
    - **路径与模板**：路径规则表 + 模板编辑器（实时预览、白名单）。
    - **敏感操作**：三个清除入口。
4. 按文件覆盖：frontmatter 写 `obsidian-auto-headings: true/false` 可强制开 / 关单个文件；「立即重新编号」命令绕过一切开关。

## 深入定制

默认行为不够用时，这些是你会用到的。

### 随便改标题，链接自动跟

大多数编号插件只更新数字，标题文字变了引用却不跟。本插件两者都同步。

```
<!-- note.md，改动前 -->
## 1 Foo

<!-- other.md -->
详见 [[note#Foo]]。
```

你改了标题——不只是数字，连文字一起改：

```
<!-- note.md，改动后 -->
## 2 Foobar
```

引用在同一次编辑里自动跟着变：

```
<!-- other.md -->
详见 [[note#Foobar]]。
```

不用手动全库查找替换，锚点不断。这套做法承袭自 Header Enhancer——目前唯一解决这个痛点的插件——并在其基础上做了几处针对性改进：原子写入（避免改写中途被打断导致文件半损坏）、更稳妥地处理重复标题，等等。（少数边界情况——重复标题、块引用、多级锚点——出于安全考虑不处理，见[说明](#说明)。）

默认开启，首次实际改写链接时会弹一次说明性提示。如果你想自己管理链接，也可以在**设置 → 全局设置**里关掉。

### 完全可定制的模板

每一级标题（H1–H6）都能独立设置：

-   **前缀**与**后缀**（如"第"+"章"→`第1章`）
-   **序号样式**——七种：阿拉伯数字（`1`）、中文数字（一二三）、带圈数字（①②③）、大小写字母、大小写罗马数字
-   **序号间隔符**（`1.1` 中的 `.`）与**标题间隔符**（数字与标题文字之间的空格）
-   **继承前级**开关——默认开启（`1.1.1`）；关闭某一级后该级只显示自己的序号（如 `a)` 而非 `1.a)`）
-   **祖先序号渲染**——解决的是两种排版惯例之间一个真实的冲突。提纲式希望每个祖先段被嵌进更深前缀里时仍保留**自身**的序号字形（`1.a.①`）；中文书式恰恰相反——章节标题本身要显示"一"，但它一旦作为祖先段出现在更深的节号里就该降级为阿拉伯数字（H2 显示"一"，到 H3 变成 `1.1`）。单一的序号字段没法同时满足这两个方向，所以 `ancestorNumeral` 只控制祖先段，与本级自身怎么呈现互不影响。

以上所有设置都会实时预览，写入文件前就能看到确切的编号格式。可以新建任意多个具名模板（在设置面板里改名、编辑或删除）。

### 一个文件夹一个模板——甚至一个文件一个模板

路径规则表把文件夹和单个文件映射到具名模板，**最具体的规则优先**（文件级规则会覆盖它所在文件夹的规则）。`/papers` 用学术编号，`/book` 用章节样式，其余地方用默认的 `1 / 1.1`——同一个库里互不干扰，无需手动切换。规则表支持拖拽排序、根据库里真实路径自动补全，并在"全局自动编号开着却没设根规则 `/`"时提醒你。

### 白名单——把结构性标题排除在计数之外

"目录""附录""参考文献"这类标题不该有编号，也不该占用计数器的槽位。白名单（按模板配置）用三种匹配方式处理：

-   **全部匹配**——标题文字与条目完全相同
-   **部分匹配**——标题文字包含该条目
-   **子树匹配**——命中的标题**及其下所有子标题**整体作为一个块被豁免；这个块结束后编号重新开始（很适合"附录不该打乱后面章节的计数"这种场景）

子树块结束后编号**重新开始**、而不是接着块前的数继续算，这是默认行为，不需要你另外找开关去打开——对学术与技术引用规范（APA、Chicago、IEEE、RFC、GB/T 7714）的调研发现，约 85% 的规范都在附录一类的块之后断开编号、其后重新起算。

默认模板已经预填充了常见的结构性词条，中英文都有（目录、附录、附图、附表、参考文献、致谢、摘要、索引及对应英文）。

编辑器本身也是为**快速调整**而设计的，不只是一个平铺列表：

-   **点击已有条目的文字即可原地编辑**——不用先删除再重新添加
-   匹配方式是行内的**分段控件**（`=` 全部 / `≈` 部分 / `▸` 子树）——单击切换，双语 tooltip 说明每个图标的含义
-   每条都有**命中数角标**，显示它在当前打开文件里豁免了多少个标题，悬停可看具体是哪些
-   当"全部/部分匹配"条目命中的标题**含子标题**时会出现**⚠ 警告图标**——提示你也许该换成子树匹配
-   **搜索框 + 排序下拉**（按添加顺序 / A–Z / 匹配方式）让长白名单也好管理
-   面板底部的实时预览会精确显示当前文件里哪些标题正在被豁免

### 清除命令：需要清空重来时

-   **立即重新编号**——立刻强制重编当前文件（见[开箱即用](#开箱即用)——这会绕过一切开关）
-   **清除当前文件编号**——剥掉本插件曾经写过（或可能写过）的全部编号前缀，还原为裸标题
-   **清理非本插件的标题编号**——只剥离**不是**本插件写的编号（手写的 `1.` 前缀、导入文档自带的编号等），本插件自己写的编号保持不动——用于接管你不是从零开始写的文档
-   **清除全库编号**——设置面板里的一个按钮，刻意**不做成命令**（避免被快捷键或命令面板误触发），需要二次确认，且收纳在默认折叠的"危险区域"里

## 工作原理——以及一件你应该知道的事

为了把自己写的编号与你的正文区分开，插件在每个编号前缀末尾写入一个不可见的 **Word Joiner** 字符（U+2060）：`## 1 ⁠我的标题`。这正是安全性的来源——只是*看起来*像编号的正文（`2024 年度总结`、`API 设计`）不会被误当旧编号吃掉。

用不可见 Unicode 字符标记前缀这个思路，最初来自 gurjar1/auto-heading-obsidian。本插件在此之上又加了一层保险：自 0.7.20 起，哨兵字符不再只放在前缀末尾，而是**首尾各放一个**。如果你在编辑时（比如改后缀）不小心把尾部那个删掉了，行首那个依然存活，成为"这里确实有插件前缀"的证据——于是下一次重新编号能把这一行**自愈**回正确状态，而不是要么把序号叠加重复、要么瞎猜误伤你的正文。

对你的影响：

-   该字符不可见，不影响排版、导出与阅读。
-   复制 / 导出的文本会带着它；若用 `grep` 精确检索标题文本，注意它位于序号与标题之间。
-   若手动删编号残留了零散字符，「清除当前文件编号」/「清理非本插件编号」命令可一键清理。

## 安装

**社区插件市场**（一旦通过审核）：设置 → 社区插件 → 浏览 → 搜索"Auto Headings"。

**手动安装**：从[最新 Release](../../releases/latest) 下载 `main.js`、`manifest.json`、`styles.css` 放入 `<库>/.obsidian/plugins/auto-headings/`，重载 Obsidian 后启用。

## 命令

| 命令                   | 作用                                           |
| ---------------------- | ---------------------------------------------- |
| 立即重新编号           | 立刻重编当前文件，绕过一切开关                 |
| 清除当前文件编号       | 剥离全部编号前缀（含手写样式）                 |
| 清理非本插件的标题编号 | 只剥**非**本插件写入的编号——用于接管导入的文档 |

## 说明

-   **语言**：插件界面自动跟随 Obsidian 语言（中 / 英），也可在设置中锁定。本 README 有[英文版](README.md)。
-   **Backlink 同步的边界**：为避免误改，以下情况保守跳过——重复标题（同名标题多处）、块引用（`^id`）、多级锚点（`#A#B`）。开启同步不会回溯修复开启之前就已断的链接。也可在**设置 → 全局设置**中关闭。
-   **撤销**：单文件改写是一次编辑器事务，一次 `Ctrl/Cmd+Z` 即可撤销；对*其他*文件的 Backlink 改动不在该事务内。「清除全库」**不在**撤销历史内，请先备份。
-   **移动端**：支持（`isDesktopOnly: false`）。
