## K：按路径把不同文件夹映射到不同模板

**步骤：**

1. 设置面板「模板」区新建一个模板（如「技术文档」），随意改其格式（如 H2 用中文「一」）。
2. 「路径规则」区点 **[+ 添加规则]**，路径模式填一个真实文件夹（输入框有补全，选 `Projects/`），
   模板下拉选「技术文档」。确保上方仍有一条 `/` → 「默认」根规则。
3. 在 `Projects/` 下任意新建笔记，粘贴下面内容，停顿等自动编号 → 应按「技术文档」（中文）编号。
4. 在 `Projects/` **之外**新建笔记，粘贴同样内容 → 应按「默认」（阿拉伯）编号。

**预期：** 文件夹内用专属模板、文件夹外用默认模板（最具体的规则优先，`/` 根规则兜底）。

```md
## 概述
### 细节
```

## K10：删掉 `/` 根规则 → 兜底缺失提示条

**步骤：** 在「路径规则」区把 `/` 根规则整行删除（点其右侧 ✕）。

**预期：** 当「全局自动编号」为开时，面板顶部出现提示条
「⚠ 无根路径规则（/）…」与 **[+ 添加 / 根规则]** 快捷按钮；此后不在任何更具体规则内的文件
**不再自动编号**（点快捷按钮可一键恢复）。

---

## I3：全局自动编号关，但某文件 frontmatter 强制 ON

**步骤：** 关闭「全局自动编号」开关 → 大多数文件不再自动编号。给下面这篇加 `obsidian-auto-headings: ON`。

**预期：** 这一篇**仍**随编辑自动编号（文件级强制 opt-in 覆盖全局关）。

```md
---
obsidian-auto-headings: ON
---

## 强制编号的章节
```

---

## I6：手动命令绕过 OFF 与全局开关

**步骤：** 对下面这篇（frontmatter `OFF`、或在全局开关关时）执行命令
**「立即重新编号（当前文件）」**。

**预期：** **照常编号**——手动命令绕过全局开关与 `OFF`，仅在「命不中任何路径规则」时才弹
Notice「当前文件未匹配任何路径规则，无法编号」。

```md
---
obsidian-auto-headings: OFF
---

## 手动也能编号
```

---

## K9：删除被路径规则引用的模板 → 知情确认对话框

**步骤：** 删除上面建的「技术文档」模板（它正被 `Projects/` 规则引用）。

**预期：** 弹出对话框，列出受影响的规则（`Projects/`），并让你选择删除后这些规则改用
**默认**（缺省）/ 其他模板 / **删除这些规则**；确认后按所选处理，绝不留下指向已删模板的悬空引用。
（删模板**不会**回滚已编号文件——已有编号冻结现状，须用 M6 的「清除编号」才会抹掉。）
