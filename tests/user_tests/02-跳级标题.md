# 场景 02：跳级标题（缺失中间级的占位策略）

当标题层级跳跃（如 H3 后直接跟 H5，中间没有 H4），缺失的中间级别如何呈现**由模板的
「跳级缺失层级」设置决定**（设置面板 → 展开该模板 → 底部）：

- **补位**（默认）：保留一段并填入「占位字符」，使序号段数等于标题深度（H_n 有 n−1 段）。
  占位字符**仅限数字**——`0`（默认，得 `1.1.0.1`）或 `1`（得 `1.1.1.1`）。
  （限数字是为保证编号始终能被干净剥离、不重复叠加。）
- **不补位**：省略该段，H5 会呈现为三段、与 H4 同形（`1.1.1`）。

无论补不补，该级计数器本身仍保持 0，**随后首个真实的该级标题仍从 1 起算**，不被借号。

下面以**默认模板（补位、占位字符 `0`）**为例。若你把占位字符改成 `1`，则 `1.1.0.1` 处会变成
`1.1.1.1`；若选「不补位」，则变成 `1.1.1`。

**默认模板下的预期结果**：

- `## 章` → `1`
- `### 节` → `1.1`
- `##### 细目甲` → `1.1.0.1`（H5 直接跟在 H3 后，缺失的 H4 显示为 `0`，共四段）
- `##### 细目乙` → `1.1.0.2`
- `#### 子节` → `1.1.1`（首个真实 H4，从 1 起算，未被前面跳级借号）
- `###### 末级` → `1.1.1.0.1`（H6 跟在 H4 后，缺失的 H5 显示为 `0`，共五段）

对照：若中间级别**真实存在**，则不会出现 `0`——例如先有 `#### 子节`(`1.1.1`)，其下的
`##### 细目` 就是 `1.1.1.1` 而非 `1.1.0.1`。

**可复制内容**：

```md
# 跳级测试文档

## 章

### 节

##### 细目甲

##### 细目乙

#### 子节

###### 末级
```
