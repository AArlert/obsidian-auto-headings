# 场景 03：多个一级标题（起始编号层级 topLevel）

插件**永不改写标题的 `#` 层级**（不再有"把多余 H1 降级成 H2"那套）。多个 H1 怎么编号，完全由
模板的**「起始编号层级」（topLevel，默认 H2）** 决定（设置面板 → 展开模板 → 顶部下拉）：

- 比 topLevel 浅的标题：不编号、原样保留，但仍是计数器**重置边界**（其后更深标题重新从 1 起）。
- topLevel 及更深的标题：正常编号，以 topLevel 为序号第一段。

## 子场景 A：默认 topLevel=H2（多个 H1 都不编号，各自重置 H2）

**预期结果**：

- `# 第一篇` → 原样（H1 不编号）
- `## 概述` → `1`
- `## 细节` → `2`
- `# 第二篇` → 原样（重置边界）
- `## 总结` → `1`  ← 第二篇的 H2 重新从 1 起

**可复制内容**：

```md
# 第一篇

## 概述

## 细节

# 第二篇

## 总结
```

## 子场景 B：topLevel=H1（所有 H1 都编号）

把模板「起始编号层级」改为 **H1** 后，同样的内容预期变为：

- `# 第一篇` → `1`
- `## 概述` → `1.1`
- `## 细节` → `1.2`
- `# 第二篇` → `2`
- `## 总结` → `2.1`

## 子场景 C：首个 H1 作标题、其余 H1 要编号

设 topLevel=H1，再把那一行标题（如「我的文档」）加入**白名单**（M4 上线后），即可让标题不编号、
其余 H1 从 1 开始。M4 未上线前，topLevel=H1 会把首个 H1 也编成 `1`。
