# 用户手动测试用例（user_tests）

> 场景的**总清单与预期真值表**（含已知 bug 状态）在 [`../../doc/testplan.md`](../../doc/testplan.md)；
> 本目录是它"user"场景的可复制实测载体，每个 `.md` 都应能追溯回 testplan 的某条场景 ID。

本文件夹收录**面向真人、在 Obsidian 里实测**的样例文档，与 `tests/dev_tests/`（自动化单元
测试）互补：

- `tests/dev_tests/` — 由 agent 维护的 Vitest 单元测试，CI / `npm test` 跑的就是它们。
- `tests/user_tests/` — 一批 `.md` 场景文件，**给用户复制粘贴进真实 Vault 验证边界行为**。
  每个文件先用文字说明「场景 + 默认模板下的预期结果」，再把**可复制的原始内容放进 Markdown
  代码块**——这样在 Obsidian 阅读模式下，代码块右上角有「复制」按钮，可一键拷走粘贴到新笔记里实测。

## 怎么用

1. 在测试 Vault 里启用本插件（默认模板：阿拉伯数字、`.` 间隔、空格分隔）。
2. 打开下面任一场景文件，切到**阅读模式**，点代码块的复制按钮。
3. 新建一篇笔记，粘贴，观察插件自动编号的结果是否与「预期」一致。

## 后续扩展约定

随着功能上线（如**按路径区分模板**、**自定义模板样式**、**白名单匹配方式**等），在此**按场景 /
模板分目录或追加文件**：每个新文件同样遵循「先文字说明场景与预期 → 再用代码块给可复制内容」的
格式，并在上面的表格登记。涉及特定模板的场景，要在说明里写清该用哪个模板、各级样式如何配置。
