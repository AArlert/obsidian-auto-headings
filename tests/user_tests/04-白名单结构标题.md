# 场景 04：白名单结构性标题（Milestone 4，已上线 0.4.0）

> **当前状态**：白名单匹配已接入插件主流程。默认模板预填充了中英常用结构性词表
> （目录 / 附录 / 附图 / 附表 / 参考文献 / 致谢 / 摘要 / 索引 及其英文，默认「全部匹配」），
> 复制下面内容到启用了插件的 Vault，停顿防抖延迟后即应看到结构性标题不被编号。

设计目标：命中白名单的结构性标题（如 `目录`、`附录`、`参考文献`）**不写编号前缀，且不占用计数器槽位**
（计数器既不累加也不归零，因此不产生跳号）。命中判定经归一化（去行内格式 + NFKC + 折叠空白 + 大小写不敏感）。

## 一、基础命中（默认词表，全部匹配）

**预期结果**：

- `## 目录` → 不编号（命中白名单）
- `## 正文` → `1`（`目录` 未占槽位，故正文仍是 1 而非 2）
- `### 小节` → `1.1`
- `## 参考文献` → 不编号（命中白名单）

**可复制内容**：

```md
# 带结构性标题的文档

## 目录

## 正文

### 小节

## 参考文献
```

## 二、归一化健壮性（仍应命中「目录」）

加粗、全角空格、大小写都不影响命中（仅用于判定，不改写你的文本）：

```md
# 归一化

## **目录**

## 正文一

## APPENDIX
```

**预期**：`**目录**` 与 `APPENDIX`（命中默认词表 `Appendix`，大小写不敏感）都不编号；`正文一` → `1`。

## 三、三种匹配方式（在设置面板的模板白名单编辑器里改 `附录` 的匹配方式实测）

- **全部**（默认）：仅 `## 附录` 自身豁免；`## 附录 A`、`## 附录` 下的 `### 数据表` 仍被编号
  （此时面板该条目会显示 ⚠，提示「含子标题，建议改用子树」）。
- **部分**：`## 附录`、`## 附录 A`、`## 技术附录` 都豁免（凡含「附录」）。
- **子树**：`## 附录` 及其下属 `### 数据表` / `#### 明细` 整块豁免、均不占槽位。

```md
# 匹配方式对比

## 概述

## 附录

### 数据表

#### 明细

## 附录 A

## 小结
```

**子树匹配预期**：`概述` → `1`、`附录`/`数据表`/`明细` 全部不编号、`附录 A` 不编号（若「附录」为部分匹配）
或编号（若为全部/子树——子树以**精确命中**为根，`附录 A` ≠ `附录` 故不在根集）、`小结` → `2`。

> 在设置 → Auto Headings → 展开模板编辑面板底部的「白名单」区即可增删条目、切换匹配方式，
> 面板会针对**当前打开的文件**实时显示「将豁免 N 个标题：…」与逐条命中数 / ⚠ 角标。
