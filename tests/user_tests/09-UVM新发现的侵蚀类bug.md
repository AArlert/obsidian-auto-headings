# 场景 09：0.6.2 UVM 发现的「侵蚀 / 吞标题」类 bug（已全修，作复验样例）

> 对应 [`doc/testplan.md`](../../doc/testplan.md) **§3.3 U1/U2/U3** 与 **C6 / B10 / E12**。这些由升级后的
> UVM 框架（explore 模式 + 幂等性记分板）撞出，**已于 0.6.3（U1/U2）与 0.6.6 方案A（U3 根治）修复**。
> 下面各小节标注的「当前（错误）现象」是**修复前**的历史行为——如今照步骤实测应**全部不再出现**：
> 数字 / 字母起头的正文（`2024 总结`、`API 设计`）会被完整保留且幂等。若复现出旧现象即为回归。

---

## U1：低于起始编号层级（topLevel）的标题被「逐次蚕食」（最严重）

**前提：** 把模板的**起始编号层级**设为 **H3**（即只给 H3 及更深的编号，H1/H2 不编号）。

**步骤：**

1. 新建笔记，粘贴下面内容。其中 H2 标题文本是 `1 2024 总结`（比如你先前手写了序号、又写了年份）。
2. 触发一次自动编号（敲一下）。观察 H2 行。
3. **再触发一次**（再敲一下）。再观察。
4. 再触发若干次。

**当前（错误）现象：** 每触发一次，H2 行左边就被吃掉一段——
`## 1 2024 总结` → `## 2024 总结` → `## 总结` → …（直到没有「数字+空格」可剥）。
即**每次输入停顿（自动编号）都在悄悄删你的标题文字**。

**预期（修复后）：** 应当**幂等**——要么稳定保留 `## 1 2024 总结`，要么一次到定点后不再变化，
绝不逐次蚕食。

```md
# 篇（不编号）
## 1 2024 总结
### 细节
```

> 根因（见 testplan §3.2 U1）：低于 topLevel 的标题走「剥一层但不重新写前缀」的分支，没有「补回前缀」
> 这一步去形成定点，于是反复剥下去。计数范围内的标题因为剥完会补回前缀、能停在定点，所以不受影响。

---

## U2：标题间隔符设成标点（如「。」）时，标题开头的数字被吞（与 E5b 承诺冲突）

**步骤：**

1. 把某级**标题间隔符**设成 **`。`**（句号，而非默认空格）。
2. 新建笔记，写一个该级标题，文本以年份开头，如 `## 1。2024 总结`（或先编号再手动补回 `2024`）。
3. 触发。

**当前（错误）现象：** `## 1。2024 总结` → `## 1。总结`——你的 `2024` 被当成又一段序号吃掉了。
（注意：若标题间隔符是**空格**，`## 1 2024 总结` 会被**稳定保留**——这正是 E5b 承诺，见场景 07。
bug 在于：换成标点间隔符后该承诺失效。）

```md
## 1。2024 总结
```

---

## U3：用字母序号（a/A）时，英文起头的标题被吞（E5「数字起头被吃」的字母版）

**步骤：**

1. 把某级**序号样式**设成 **小写字母** 或 **大写字母**。
2. 新建笔记，写一个以英文词起头的该级标题，如 `## API 设计`。
3. 触发。

**当前现象：** `## API 设计` → `## A 设计`——`API` 被当成字母序号剥掉、再编号成 `A`。

**说明：** 这是既有取舍 L1/E5 的字母版（字母样式启用时，剥离器会把字母开头当序号）。一次性、幂等。
列此备案，提醒：**给英文起头的标题用字母序号需谨慎**。

```md
## API 设计
```
